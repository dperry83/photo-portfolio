# Photography Website Planning Document

This document is the working reference for version 1 of the photography website.

Version 1 is intentionally a static front end built with HTML, CSS, and vanilla JavaScript. The goal is to build a simple, clear, founder-focused portfolio site that feels credible, calm, and low-drama while strengthening core web fundamentals through semantic structure, CSS styling, and light browser-based interaction.

## Planning goal

This document exists to keep the project small, focused, and easy to reference during the build. It is meant to capture the core planning decisions and working standards for version 1.

## Project rules

- Keep version 1 small and static.
- Every page should have one clear job.
- Every image on the site should have a reason to be there.
- Copy should be plain, founder-aware, and low-jargon.
- Design should feel credible, calm, modern, and human.
- JavaScript should enhance the experience, not carry it.

## Version 1 sitemap

- Home
- Portfolio / Gallery
- About
- Contact

This keeps the scope focused on the core portfolio experience and avoids adding extra pages before the site fundamentals are solid.

## Page goals

| Page | Main job | What success looks like |
|------|-----------|-------------------------|
| Home | Explain who the site is for, what problem is being solved, and why it matters | A founder quickly understands the offer and feels curious enough to keep going or reach out  |
| Portfolio / Gallery | Show strong, useful examples of the work | The work feels credible, modern, and relevant to founder use cases rather than generic portrait photography  |
| About | Build trust and reduce fear about the process | A camera-shy founder feels understood, guided, and less worried about awkward photos  |
| Contact | Make the next step feel small and low-pressure | Reaching out feels simple, guided, and clear  |

## Design system planning

### Needed decisions

- Font pairing or font system
- Core color palette
- Spacing rhythm
- Button styles
- Link styles
- Container widths
- Image treatment rules
- Basic card / content block styles

### Design standard

The visual system should feel:

- Credible
- Calm
- Modern
- Human
- Low-drama

It should avoid:

- Generic corporate stiffness
- Trendy startup gradients
- Moody luxury-photo clichés
- Visual noise that competes with the work

## UX priorities

- Fast understanding of who the site is for
- Confidence that the photographer understands founder needs
- Low-friction movement through the site
- Strong portfolio browsing experience
- Simple path to contact

Because semantic HTML is foundational for structure and accessibility, page layout and content hierarchy should be planned with real headings, sections, navigation, and supporting text in mind from the start.

## JavaScript planning

Version 1 enhancement targets:

- Menu toggle
- Gallery filtering
- Lightbox behavior
- Simple contact form behavior

This keeps the site usable even before JavaScript is layered on, which matches the progressive-enhancement mindset for a static front end.

## Definition of done for version 1

Version 1 should be considered ready when:

- All four pages are built and linked
- The design system feels consistent
- The portfolio experience is strong and easy to navigate
- Contact is clear and low-friction
- The site works well on mobile and desktop
- Basic accessibility and semantic structure are in place
- JavaScript enhancements are helpful, not necessary for basic use
