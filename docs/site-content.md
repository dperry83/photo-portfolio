# Photography Website Content Document

This document holds the content-facing structure for version 1 of the photography website.

It separates page structure and content needs from the broader planning document so the project is easier to reference during writing, layout, and build work.

## Page outlines

### Home

The homepage should do the most strategic work in the smallest amount of space.

Suggested sections:

1. Hero
2. Short credibility-gap message
3. Selected work preview
4. What these photos are for
5. Simple process
6. Reassurance / trust
7. Contact call to action

### Portfolio / Gallery

This page should show proof without turning into a giant image dump.

Suggested sections:

1. Intro / framing text
2. Filter controls
3. Gallery grid
4. Lightbox interaction
5. Contact call to action

### About

This page should help the visitor understand the person, the perspective, and the working style behind the site.

Suggested sections:

1. Personal intro
2. Why founders
3. How the process works
4. Reassurance for non-models
5. Contact call to actio

### Contact

This page should remove the last bit of friction around reaching out.

Suggested sections:

1. Short intro
2. What to send or what happens next
3. Contact form or simple contact path
4. Final reassurance

## Content needs

Minimum useful content list for version 1:

- Homepage headline
- Homepage subhead
- Short founder-focused positioning copy
- Intro copy for the portfolio page
- About-page biography / positioning copy
- Process explanation
- Contact-page intro copy
- Call-to-action language
- Portfolio image shortlist
- Basic alt text and file naming plan

## Content note

This document is for page structure and written-content planning. The main planning document remains focused on project rules, UX priorities, design-system decisions, and definition of done.
